# 프로젝트 구조 및 스타일링 아키텍처

이 문서는 `yamyam-fe` Vue.js 프로젝트의 전체 구조와 CSS 스타일링 방식에 대한 분석 내용을 담고 있습니다.

## 1. 디렉토리 구조

프로젝트는 Vite를 기반으로 하는 표준 Vue 3 구조를 따릅니다.

- **`/` (Root)**
  - `package.json`: 프로젝트의 의존성(Vue, Vue Router, Axios 등) 및 스크립트를 관리합니다.
  - `vite.config.js`: Vite 빌드 설정을 담당합니다. 현재는 기본적인 Vue 플러그인과 `@` 경로 별칭만 설정되어 있습니다.
  - `index.html`: 애플리케이션의 기본 HTML 진입점입니다.

- **`/src`**: 핵심 소스 코드가 위치합니다.
  - `main.js`: Vue 애플리케이션의 메인 진입점입니다. Vue 인스턴스를 생성하고, 라우터와 전역 스타일을 등록합니다.
  - `App.vue`: 최상위 루트 컴포넌트입니다. 모든 뷰는 이 컴포넌트의 `<router-view />`를 통해 렌더링됩니다.
  - `style.css`: 프로젝트 전반에 적용되는 글로벌 CSS 스타일 파일입니다.

- **`/src/api`**: 백엔드 API와의 통신을 담당하는 모듈이 위치합니다.
  - `axios.js`: 기본 `axios` 인스턴스 설정이 포함될 수 있습니다.
  - `diet.js`, `food.js`: 각 도메인 별 API 요청 함수들을 정의합니다.

- **`/src/assets`**: 이미지, 폰트 등 정적 에셋 파일이 위치합니다.

- **`/src/components`**: 여러 뷰(View)에서 재사용될 수 있는 공통 컴포넌트가 위치합니다. (예: `AppHeader.vue`)

- **`/src/constants`**: 프로젝트 전역에서 사용되는 상수 값을 정의합니다. (예: `meal.js`)

- **`/src/router`**: Vue Router 설정을 담당합니다.
  - `index.js`: 라우트와 뷰 컴포넌트를 매핑합니다.

- **`/src/utils`**: 날짜 포맷팅, 영양소 계산 등 순수 함수 유틸리티 모듈이 위치합니다.

- **`/src/views`**: 각 라우트에 해당하는 페이지 레벨의 컴포넌트(뷰)가 위치합니다.

## 2. CSS 스타일링 아키텍처

- **전역 스타일 (Global Styling)**:
  - `src/style.css` 파일은 `src/main.js`에서 직접 `import`되어 전역으로 적용됩니다.
  - `App.vue` 컴포넌트 내의 `<style>` 태그 또한 `scoped` 속성이 없으므로 전역적으로 적용됩니다.
  - 이 두 곳에서 애플리케이션의 기본 배경색, 폰트, CSS 리셋 (`* { margin: 0; padding: 0; }`) 등을 정의하고 있습니다.

- **컴포넌트 스코프 스타일 (Component-Scoped Styling)**:
  - 개별 `.vue` 파일(컴포넌트 및 뷰) 내에서 스타일을 정의할 때는 `<style scoped>` 블록을 사용하는 것이 표준 방식입니다.
  - 이는 다른 컴포넌트와의 CSS 충돌을 방지하며, 해당 컴포넌트의 DOM 요소에만 스타일을 적용합니다.

- **CSS 프레임워크 및 라이브러리**:
  - `package.json` 분석 결과, Bootstrap, Tailwind CSS, Vuetify와 같은 대형 UI 프레임워크는 사용하고 있지 않습니다.
  - 모든 스타일은 순수 CSS(Plain CSS)를 통해 작성됩니다.
  - Vite의 기본 구성을 통해 `PostCSS`와 `autoprefixer`가 자동으로 적용되어, 최신 CSS 문법의 브라우저 호환성을 보장합니다.

## 3. 개발 가이드라인

- **컴포넌트/뷰 생성**: 새로운 페이지는 `/src/views`에, 재사용 가능한 UI 조각은 `/src/components`에 `.vue` 파일로 생성합니다.
- **스타일 작성**:
  - 여러 페이지에 공통적으로 적용되어야 하는 스타일은 `src/style.css`에 추가합니다.
  - 특정 컴포넌트나 뷰에만 한정되는 스타일은 해당 파일의 `<style scoped>` 내에 작성하는 것을 원칙으로 합니다.
- **API 연동**: 백엔드와의 통신 로직은 `/src/api` 디렉토리 내에 도메인별로 분리하여 작성합니다.
